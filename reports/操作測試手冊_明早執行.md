# Stock Journal 明早一鍵驗收操作手冊

## 0) 啟動
1. 啟動服務：`python server.py`
2. 確認首頁可開啟：`http://localhost:8000`
3. 以 Google 登入 A 帳號（owner）。

## 1) 建立基礎資料（A 帳號）
1. 新增帳戶：主帳戶-A
2. 新增 2 筆交易（不同股票）
3. 驗證 `/api/accounts`、`/api/trades` 皆可看到資料

## 2) 多使用者隔離測試（B 帳號）
1. 登出後登入 B 帳號
2. 開啟交易/帳戶清單，預期看不到 A 的資料
3. B 自行新增 1 筆交易，回到 A 帳號確認看不到 B 交易

## 3) 協作權限功能（新）
1. A 帳號呼叫 API 新增授權：
   `POST /api/permissions` body:
   `{ "collaborator_uid": "<B_UID>", "role": "viewer" }`
2. 查詢授權清單：`GET /api/permissions`
3. 改成 editor 再測一次，確認可更新/新增。

## 4) 稽核日誌（新）
1. A 進行新增交易、修改交易、刪除交易
2. 查詢 `GET /api/audit?limit=50`
3. 預期可看到 `trade.create/trade.update/trade.delete` 與時間戳

## 5) 不可變交易事件帳（新）
1. 每次交易新增/修改都會寫入 `trade_events`
2. 驗證方式（本機）:
   `sqlite3 data/stock_journal.db "select trade_id,event_type,created_at from trade_events order by id desc limit 20;"`

## 6) 高風險操作防呆（新）
1. 刪除交易不帶 `X-Confirm: YES`，預期失敗
2. 帶 `X-Confirm: YES` 重試，預期成功
3. 匯入覆蓋 `/api/trades/import` 不帶 `confirm=YES`，預期失敗

## 7) 備份還原（新）
1. 建立備份：`./scripts/backup_restore.sh backup`
2. 列備份：`./scripts/backup_restore.sh list`
3. 還原：`./scripts/backup_restore.sh restore backups/stock_journal_YYYYMMDD_HHMMSS.sqlite3`

## 8) 異常警示（本階段）
1. 目前版本先以 audit + server error log 為警示來源
2. 明天可先檢查：
   - `/api/audit` 是否有異常大量刪除/匯入
   - server console 是否有 traceback
